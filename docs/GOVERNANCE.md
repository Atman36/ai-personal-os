# Governance

## Deny by default

If the OS is unsure whether an action is allowed, it should not do it. It should ask for approval or route to Governor.

## Human-only zones

- Legal commitments.
- Financial commitments.
- External sends.
- Public claims.
- Credential/security changes.
- Sensitive personal/customer data handling.
- Destructive operations.

## Promotion of autonomy

A workflow can move from A1 to A2/A3 only after:

1. it has repeated successful runs;
2. acceptance criteria are stable;
3. failures are understood;
4. Governor approves the boundary;
5. user explicitly approves.

## Audit

The OS should preserve:

- decision briefs;
- task state changes;
- approval notes;
- validation results;
- weekly review summaries.
